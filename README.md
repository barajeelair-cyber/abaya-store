# عبايات أمل  —  متجر العبايات

متجر إلكتروني فاخر متكامل بـ **HTML + CSS + Vanilla JS**. يفتح مباشرة في المتصفح بدون أي خطوة build.

## أين تضعين اللوجو؟

ضعي ملف اللوجو في:
```
assets/img/logo.png
```
سيظهر تلقائياً في:
- شريط واجهة الزبون (المتجر)
- الشريط الجانبي للإدارة
- شاشة تسجيل الدخول

> المقاس المستحسن: 200×200 بكسل، PNG بخلفية شفافة.

## بيانات الدخول الافتراضية

```
المستخدم: admin
الكلمة:   admin123
```
يمكن تغييرها من **الإعدادات → تغيير كلمة المرور** داخل اللوحة.

## ملفات المشروع

```
abaya-store/
├── index.html                ← واجهة الزبون
├── admin.html                ← لوحة الإدارة (مع شاشة دخول)
├── README.md
└── assets/
    ├── img/
    │   └── logo.png          ← ضعي اللوجو هنا
    ├── css/
    │   ├── customer.css
    │   └── admin.css
    └── js/
        ├── data.js           ← طبقة البيانات (ProductsAPI, OrdersAPI, SettingsAPI, AuthAPI)
        ├── customer.js
        └── admin.js
```

## ✨ المزايا الجديدة

### واجهة الزبون
- 🎨 شعار + اسم المتجر بتدرج ذهبي
- 📞 شريط تواصل علوي (هاتف + واتساب)
- 🏷️ **تصنيفات**: عبايات عملية · مناسبات · سوداء · مفتوحة
- 💰 عرض **الخصم** على المنتج (سعر مشطوب + شارة خضراء)
- 🎨 لكل لون **صورته الخاصة**  —  اختيار اللون يبدّل الصورة فوراً
- 📏 المقاسات تُعطَّل تلقائياً إذا كانت غير متوفرة لهذا اللون
- 🛒 زر "عربة التسوق" بالنص + الأيقونة + شارة العدد
- 💳 شاشة الدفع تعرض **كل الحسابات البنكية** + زر نسخ
- 📸 **رفع صورة التحويل إلزامي** قبل تأكيد الطلب (سحب وإفلات أو نقر)
- ✅ بعد الإرسال: رسالة "بانتظار تأكيد الدفع"

### لوحة الإدارة
- 🔐 شاشة تسجيل دخول
- 📊 لوحة متابعة (إجمالي المنتجات، طلبات بانتظار الدفع، تنبيهات المخزون، المبيعات)
- 👗 إضافة/تعديل منتج بـ:
  - تصنيف + سعر + **خصم %**
  - **ألوان متعددة، لكل لون صورة** (drag-and-drop)
  - **جدول مخزون كامل**: كل خلية = (لون × مقاس) بمخزون مستقل
- 📦 صفحة المخزون تعرض جدول التركيبات لكل منتج منخفض + تعديل سريع
- 🧾 الطلبات: فلترة بالمدينة والحالة، عرض **صورة التحويل**، زر **"تأكيد الدفع"** سريع
- ⚙️ صفحة إعدادات شاملة:
  - معلومات التواصل (هاتف + واتساب)
  - تعديل عنوان (Headline) الصفحة الرئيسية
  - إدارة الحسابات البنكية (إضافة/تعديل/حذف بـ IBAN)
  - تغيير كلمة مرور الأدمن

## دورة الطلب الكاملة

```
[الزبونة] تختار → ترفع صورة التحويل → تؤكد الطلب
                                          ↓
                              ┌─ بانتظار تأكيد الدفع ─┐
                              │ (الإدارة تراجع الإيصال)│
                              └───────────┬───────────┘
                                          ↓ ضغط "تأكيد الدفع"
                                    قيد المعالجة
                                          ↓
                                     تم الشحن
                                          ↓
                                    تم التوصيل
```

## التشغيل

افتحي `index.html` مباشرة بالنقر المزدوج، أو شغّلي خادماً محلياً:

```powershell
cd C:\Users\User\abaya-store
python -m http.server 8000
```

- المتجر:   http://localhost:8000/
- الإدارة:  http://localhost:8000/admin.html

## كيفية ربط الواجهتين

كلاهما يستوردان نفس `data.js`، الذي يقرأ/يكتب في `localStorage` بمفتاح `abaya_amal_v2`.

```
admin.html ──┐
             ├─► data.js ─► localStorage (مفتاح: abaya_amal_v2)
index.html ──┘
```

عند تعديل منتج من الإدارة → يظهر فوراً في تبويب الزبون (عبر `storage event`).

## الانتقال إلى Backend حقيقي

كل عمليات القراءة/الكتابة معزولة داخل `data.js` فقط:
- `ProductsAPI` (list/get/save/remove/variantStock/finalPrice)
- `OrdersAPI` (list/add/updateStatus)
- `SettingsAPI` (get/save/addBank/updateBank/removeBank)
- `AuthAPI` (login/logout/changePassword)

استبدلي محتوى كل دالة بـ `fetch` إلى API الخاص بكِ (Node/PHP/Firebase) دون لمس باقي الكود. وحوّلي الدوال المستخدِمة إلى `async/await`.

## ملاحظات حول التخزين

- الصور المرفوعة (لوجو الألوان + إيصالات التحويل) تُخزَّن كـ **base64** في `localStorage`.
- الحجم الأقصى لكل صورة: **3 ميجا** (تحقق آلي).
- إجمالي مساحة `localStorage` ~5 ميجا. للمنتجات الكثيرة، يفضّل صور مضغوطة.
- لإعادة تهيئة كل شيء: في Console المتصفح:
  ```js
  localStorage.removeItem("abaya_amal_v2");
  location.reload();
  ```
